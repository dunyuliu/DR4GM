# DR4GM NPZ File Format Specification

This document describes the data structure and format of key NPZ files generated by the DR4GM (Dynamic Rupture for Ground Motion) processing pipeline, using EQDYNA scenario `eqdyna.0001.A.100m` as an example.

## Overview

The DR4GM pipeline generates 5 key NPZ files for each scenario:

1. **`stations.npz`** - Station metadata and locations
2. **`velocities.npz`** - Raw velocity time series data  
3. **`ground_motion_metrics.npz`** - Computed ground motion metrics per station
4. **`gm_statistics.npz`** - Distance-binned statistical analysis
5. **`geometry.npz`** - Fault geometry and source parameters

---

## 1. stations.npz - Station Metadata

**Purpose**: Contains station locations, IDs, and metadata for all stations in the scenario.

| Key | Shape | Type | Description | Sample Values |
|-----|-------|------|-------------|---------------|
| `station_ids` | (367021,) | int64 | Unique station identifiers | [0, 1, 2, 3, 4, ...] |
| `locations` | (367021, 3) | float64 | Station coordinates [x, y, z] in meters | [[19903.3, -44900.0, 0.0], [19804.2, -44900.0, 0.0], ...] |
| `chunk_ids` | (367021,) | int64 | Source data chunk IDs | [7, 7, 7, 7, 7, ...] |
| `local_ids` | (367021,) | int64 | Local station ID within chunk | [0, 1, 2, 3, 4, ...] |
| `station_types` | (367021,) | float64 | Station type indicator | [1.0, 1.0, 1.0, ...] (all surface) |
| `coordinate_units` | () | str | Coordinate system units | "m" |

**Total Stations**: 367,021  
**Coordinate System**: Post-rotation (fault runs N-S along Y-axis)

## 2. velocities.npz - Velocity Time Series

**Purpose**: Contains raw velocity time series data for all stations in strike/normal/vertical components.

| Key | Shape | Type | Description | Sample Values |
|-----|-------|------|-------------|---------------|
| `station_ids` | (367021,) | int64 | Station identifiers (matches stations.npz) | [0, 1, 2, 3, 4, ...] |
| `locations` | (367021, 3) | float64 | Station coordinates [x, y, z] | Same as stations.npz |
| `vel_strike` | (367021, 294) | float64 | Strike-parallel velocity (m/s) | Time series per station |
| `vel_normal` | (367021, 294) | float64 | Fault-normal velocity (m/s) | Time series per station |
| `vel_vertical` | (367021, 294) | float64 | Vertical velocity (m/s) | Time series per station |
| `time_steps` | (367021,) | int64 | Number of time steps per station | [294, 294, 294, ...] |
| `dt_values` | (367021,) | float64 | Time step size per station (seconds) | [0.085030, 0.085030, ...] |
| `duration` | () | float64 | Total simulation duration (seconds) | 25.0 |
| `units` | () | str | Velocity units | "m/s" |
| `coordinate_units` | () | str | Coordinate units | "m" |

**Time Series Details**: 294 time steps × 0.085030s = 25.0s duration per station

## 4. gm_statistics.npz - Distance-Binned Statistics

**Purpose**: Contains statistical analysis of ground motion metrics as a function of distance from fault.

### Distance Information

| Key | Shape | Type | Units | Description | Sample Values |
|-----|-------|------|-------|-------------|---------------|
| `rjb_distance_bins` | (60,) | float64 | m | Distance bin centers | [250, 750, 1250, 1750, 2250, ...] |
| `distance_bin_edges` | (61,) | float64 | m | Distance bin edges | [0, 500, 1000, 1500, ...] |
| `distance_units` | () | str | - | Distance units | "m" |

### Units Information

| Key | Value | Description |
|-----|-------|-------------|
| `PGA_units` | "cm/s²" | Peak Ground Acceleration units |
| `PGV_units` | "cm/s" | Peak Ground Velocity units |
| `PGD_units` | "cm" | Peak Ground Displacement units |
| `CAV_units` | "cm/s" | Cumulative Absolute Velocity units |
| `RSA_units` | "cm/s²" | Response Spectral Acceleration units |

### Statistical Arrays

Each ground motion metric has 5 statistical arrays:

| Suffix | Shape | Type | Description |
|--------|-------|------|-------------|
| `_mean` | (60,) | float64 | Geometric mean in each distance bin |
| `_std` | (60,) | float64 | Logarithmic standard deviation |
| `_min` | (60,) | float64 | Minimum value in distance bin |
| `_max` | (60,) | float64 | Maximum value in distance bin |
| `_count` | (60,) | int64 | Number of stations in distance bin |

**Example**: PGA statistics (first 5 distance bins)
- Distance bins: [0.25, 0.75, 1.25, 1.75, 2.25] km
- PGA means: [885.7, 592.5, 478.1, 411.3, 358.6] cm/s²
- PGA std: [0.340, 0.303, 0.336, 0.374, 0.396]

### Available Metrics

**Basic Metrics**: PGA, PGV, PGD, CAV

**RSA Periods**: RSA_T_0_100, RSA_T_0_125, RSA_T_0_250, RSA_T_0_333, RSA_T_0_400, RSA_T_0_500, RSA_T_0_750, RSA_T_1_000, RSA_T_1_500, RSA_T_2_000, RSA_T_2_500, RSA_T_3_000, RSA_T_5_000

## 5. geometry.npz - Fault Geometry

**Purpose**: Contains fault geometry parameters and source information.

| Key | Shape | Type | Description | Value |
|-----|-------|------|-------------|-------|
| `fault_type` | () | str | Type of faulting mechanism | "strike_slip" |
| `fault_trace_start` | (3,) | float64 | Fault trace start point [x, y, z] (m) | [0.0, -20000.0, 0.0] |
| `fault_trace_end` | (3,) | float64 | Fault trace end point [x, y, z] (m) | [0.0, 20000.0, 0.0] |
| `fault_dip` | () | float64 | Fault dip angle (degrees) | 90.0 |
| `fault_strike` | () | float64 | Fault strike angle (degrees) | 0.0 |
| `coordinate_units` | () | str | Coordinate system units | "m" |
| `fault_length` | () | float64 | Total fault length (m) | 40000.0 |
| `rotation_applied` | () | str | Coordinate rotation applied | "counterclockwise_90deg" |
| `description` | () | str | Scenario description | "EQDYNA M6.9 strike-slip fault..." |

**Fault Configuration**: N-S striking vertical fault, 40 km length, surface rupture

## Data Processing Pipeline

### File Dependencies

```
Raw Simulation Data
         ↓
1. stations.npz + velocities.npz + geometry.npz
         ↓
2. ground_motion_metrics.npz (from velocities.npz)
         ↓
3. gm_statistics.npz (from ground_motion_metrics.npz)
```

### Coordinate System

- **Original EQDYNA**: Fault runs E-W along X-axis
- **Post-rotation**: Fault runs N-S along Y-axis (90° counterclockwise rotation applied)
- **Units**: All coordinates in meters, velocities in m/s, accelerations in cm/s²

### Quality Control

- All NPZ files pass DR4GM format validation
- Consistent station counts across files (367,021 stations)
- Unit consistency enforced throughout pipeline
- Coordinate rotation applied consistently

## Usage for Research Publications

These files provide comprehensive earthquake ground motion data suitable for:

- **Attenuation relationships**: Use gm_statistics.npz for distance-dependent analysis
- **Site effects studies**: Use ground_motion_metrics.npz for per-station analysis
- **Spectral analysis**: RSA data available at 13 periods (0.1-5.0 seconds)
- **Waveform analysis**: Raw velocity time series in velocities.npz
- **Fault modeling**: Complete geometry parameters in geometry.npz

All data follows standardized NPZ format for reproducible research and cross-scenario comparisons.
